## Interaction Flow

### Download Flow
```
1. User sees file card with ⬇️ icon
   ↓
2. User taps ⬇️
   ↓
3. Action sheet appears with 2 options
   ↓
4. User selects "Save to App" or "Save to Device"
   ↓
5. Download progress overlay shows
   ↓
6. Download completes
   ↓
7. File card updates:
   - ⬇️ changes to 👁
   - "Offline" badge appears
   ↓
8. User can now tap 👁 to view
```

### View Flow
```
1. User sees file card with 👁 icon + "Offline" badge
   ↓
2. User taps 👁
   ↓
3. Based on file type:
   
   PDF/Document:
   → Modal shows "Open with App" button
   → User taps button
   → System file picker appears
   → User selects app
   → File opens in selected app
   
   Video/Audio/Image:
   → Fullscreen viewer opens
   → File displays/plays with controls
   → User interacts (play, pause, zoom, etc.)
   → User taps ✕ to close
   
   Other:
   → Unsupported viewer shows
   → Options to open externally or share
```

## Responsive Design

### Phone (< 768px)
- Single column layout
- Full-width file cards
- Compact spacing
- Bottom sheet modals

### Tablet (≥ 768px)
- Centered content (max 700px)
- Larger icons and text
- More spacing
- Centered modals

### Large Tablet (≥ 1024px)
- Centered content (max 900px)
- Even larger touch targets
- Generous spacing
- Professional appearance

## Accessibility

- ✅ Proper contrast ratios (WCAG AA)
- ✅ Touch targets ≥ 44x44 points
- ✅ Clear visual hierarchy
- ✅ Descriptive icons
- ✅ Readable fonts (14-18px)
- ✅ Status badges with icons + text
- ✅ Error messages with actions
